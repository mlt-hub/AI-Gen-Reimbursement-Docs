# COSMIC 工具系统缺陷

## 稳定性

- **`except Exception: pass` 过多** — AI 解析、配置文件加载等处异常被静默吞掉，出错时难以定位。涉及 `gen_xlsx.py`、`cosmic_llm.py`、`excel_writer.py` 等多处
- **AI 输出格式依赖强** — FPA 的 `classification_basis_index`、COSMIC 的 JSON 结构，AI 一旦偏离格式就默默跳过（`except Exception: pass`），不报错
- **JSON 清洗逻辑重复** — `cosmic_llm.py` 的 `_parse_llm_response()` 和 `gen_xlsx.py` 的 `_ai_fill_fpa()` 各有一套去除 markdown 代码块、清理 JSON 的逻辑，未统一

## 兼容性

- **WPS 渲染差异** — 合并单元格边框需四角+四边全部显式设置为黑色，Excel 只需四角；`patternType=none` 在 WPS 中渲染为浅绿色
- **仅 Windows 提示音** — 使用 `winsound.PlaySound`，Linux/Mac 平台无声
- **模板紧耦合** — xlsx 模板的行号、合并单元格范围与代码硬关联（如 `tmpl_format_row6`、第5行表头、页脚行号），模板结构变动容易导致生成失败

## 性能

- **`delete_rows` 大文件慢** — 已改用删除行逻辑，模板行数多或输出数千行时，openpyxl 的 `delete_rows` 性能明显下降
- **元数据重复解析** — `_parse_meta_md()` 每次调用都重新读取整个文件，无缓存

## 配置

- **配置分散三文件** — `system_config.yaml`、`business_rules.yaml`、`ai_system_prompts_config.yaml` 分布在项目级和用户级两处，哪个参数在哪个文件不易确定
- **`notify_sound` 读取错位** — 代码之前读的是 `business_rules.yaml`，但用户配置在 `system_config.yaml`，已修复但暴露了配置管理问题

## 其他

- **文件被占用时崩溃** — Excel/WPS 打开输出文件时重新生成会抛出 PermissionError，未捕获处理
- **`.gitignore` 未覆盖输出目录** — `AI-Outputs/`、`log/` 等目录可能被误提交
- **日志级别不一致** — 部分信息用 `print()`，部分用 `logger`，查看时混淆
