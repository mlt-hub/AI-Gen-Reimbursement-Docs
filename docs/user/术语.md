# 术语文档

## 层级术语

| 术语 | 英文 | 说明 |
|------|------|------|
| L1 | Level 1 | 一级模块，功能领域 |
| L2 | Level 2 | 二级模块，子功能 |
| L3 | Level 3 | 三级模块，叶子功能点 |
| proc | process / functional process | 功能过程，最小的功能单元 |
| CFP | Cosmic Function Point | COSMIC 功能点数 |

## 解析策略

| 术语 | 英文 | 说明 |
|------|------|------|
| 标题样式 | heading style | 按 Word 标题样式 ID 匹配段落 |
| 多级列表格式 | multi-level list format | 按 `(numId, ilvl)` 精确匹配段落 |
| 编号格式 | numbering format | 按 `numId` 且 `ilvl` 匹配有编号的段落 |
| 用户标注法 | user annotation method | 用户在 docx 中嵌入 `###` 标记定义层级 |
| 模板映射法 | mapping method | 使用 `docx_parse_mapping_rules.yaml` 中预设的映射规则 |

## 标记语法

| 标记 | 说明 | 示例 |
|------|------|------|
| `###level:strategy###` | 标记某段落的层级和匹配策略 | `###一级模块:标题样式###` |
| `###文档开始###` | 标记功能需求章节开始 | `###文档开始###` |
| `###文档结束###` | 标记功能需求章节结束 | `###文档结束###` |
| `###一级模块:策略###` | L1 标记 | `###一级模块:多级列表格式###` |
| `###二级模块:策略###` | L2 标记 | `###二级模块:标题样式###` |
| `###三级模块:策略###` | L3 标记 | `###三级模块:多级列表格式###` |
| `###功能过程:策略###` | 功能过程标记 | `###功能过程:编号格式###` |

## 编号相关

| 术语 | 英文 | 说明 |
|------|------|------|
| numId | Numbering Definition ID | 编号定义实例 ID，标识一个编号序列 |
| ilvl | Indentation Level | 列表缩进层级，0 为顶层 |
| numFmt | Numbering Format | 编号格式类型（decimal / lowerLetter / chineseCounting 等） |

## 章节检测

| 配置项 | 说明 |
|--------|------|
| `section_begin_number` | 章节开始编号，如 `'4'` |
| `section_begin_keyword` | 章节开始标题关键字，如 `'功能需求'` |
| `section_end_number` | 章节结束编号，如 `'5'` |
| `section_end_keyword` | 章节结束标题关键字，如 `'附加值调整因子说明'` |

## Docx 文件类型

| 扩展名 | 说明 |
|--------|------|
| `.docx` | Word 文档（无宏） |
| `.docm` | Word 宏启用文档（本工具不支持） |
| `~$*.docx` | Word 临时文件（自动跳过） |

## 日志格式

| 日志前缀 | 含义 |
|----------|------|
| `用户标注法:` | 从文档中的 `###` 标记提取规则 |
| `模板映射法（mapping_N）:` | 从配置文件的 mapping_N 加载规则 |
| `标记策略:` | 解析完成的模块和过程统计 |
| `章节检测:` | 章节开始/结束的检测配置 |
| `标记检测:` | 通过 `###文档开始###` 定位的章节范围 |
| `定位章节:` | 通过文本匹配定位的章节范围 |
| `层级映射已保存:` | 本次解析的层级映射写入配置文件 |
| `跳过重复映射` | 相同规则的映射已存在，不重复写入 |
| `未找到预期结果文件:` | 无对应 JSON，跳过比较 |
| `解析结果同JSON文件一致` | 解析结果与预期一致 |
| `解析结果与JSON不一致:` | 解析结果与预期有差异 |

## 映射配置文件名

| 文件 | 路径 | 说明 |
|------|------|------|
| `docx_parse_mapping_rules.yaml` | `~/.ai-gen-reimbursement-docs/` | 映射规则配置文件，自动生成 |
| `docx_parse_mapping_rules.yaml.example` | `config/` | 示例与说明 |
| `docx_expected_result.example.json` | `config/` | 预期结果示例 |

## mapping 命名

自动生成的 mapping 从 `mapping_1` 开始递增。可通过 `--mapping mapping_1` 指定使用某一组预设规则。

## 章节检测分组

`章节检测` 支持多组命名，通过 `--chapter-detection <组名>` 指定，默认为 `default`。
