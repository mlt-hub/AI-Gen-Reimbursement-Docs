# CLAUDE 配置文件说明

Claude Code 支持三个层级的配置文件，用于控制助手的行为、权限和项目理解。

## 层级结构

| 级别 | 路径 | 提交 git | 作用范围 | 用途 |
|------|------|----------|----------|------|
| **项目级** | `{{project_root}}/CLAUDE.md` | 是 | 当前项目 | 项目结构、核心逻辑、规则 |
| **全局级** | `~/.claude/CLAUDE.md` | 否 | 所有项目 | 用户通用偏好、全局规则 |
| **本地私有** | `{{project_root}}/.claude/settings.local.json` | 否 | 当前项目 | 命令权限、本地配置 |

## 优先级

`settings.local.json` > `CLAUDE.md`（项目级） > `~/.claude/CLAUDE.md`（全局级）

本地私有配置的权限允许会覆盖全局和项目级的限制。

## 项目级 CLAUDE.md

位于项目根目录，随项目一起提交到 git。内容通常包括：

- 项目概述与目标
- 核心文件列表与说明
- 关键业务逻辑
- CLI 参数说明
- 代码风格约定
- 依赖信息

```markdown
# 项目名称

## 项目概述
一句话描述项目做什么。

## 核心文件
| 文件 | 说明 |
|------|------|

## 关键逻辑
- 解析流程
- 重要函数

## CLI 参数
| 参数 | 说明 |
|------|------|

## 代码风格
- 命名规范
- 注释要求
```

## 全局级 CLAUDE.md

位于 `~/.claude/CLAUDE.md`，对所有项目生效。内容通常包括：

- 通用规则（输出语言、编辑习惯等）
- 敏感信息提醒
- Git 操作规范
- 常用命令

```markdown
# 用户全局配置

## 通用规则
- 中文输出
- 修改前先读文件
- 不提交敏感信息
```

## 本地私有配置

位于 `.claude/settings.local.json`，用于权限白名单和本地覆盖，不提交到 git。

```json
{
  "permissions": {
    "allow": [
      "Bash(命令模式)"
    ]
  },
  "_comment": "本地私有配置"
}
```
