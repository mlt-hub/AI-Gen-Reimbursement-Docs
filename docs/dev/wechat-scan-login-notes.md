# 微信扫码登录接入说明

本文记录本项目接入微信扫码登录前需要确认的外部条件、适用边界和推荐实现策略。

## 结论

微信扫码登录可以作为远程服务模式的可选登录方式，但不应作为当前默认登录方案。

原因：

- 微信 PC 网站扫码登录通常走微信开放平台“网站应用”能力，需要创建网站应用、通过审核，并取得 `AppID` / `AppSecret`。
- 网站应用需要配置授权回调域。正式接入时应使用公网可访问域名，`localhost`、内网 IP 或临时隧道不适合作为生产回调地址。
- 个人主体是否可成功申请，以微信开放平台当前后台规则和审核结果为准。实践上，没有企业或个体工商户主体、可访问网站和合规域名时，申请成功存在较高不确定性。
- 中国大陆部署场景下，域名备案、网站可访问性、主体一致性等因素可能影响网站应用审核。

## 本项目推荐策略

短期不替换现有账号密码登录。

如后续接入微信扫码登录，应采用“配置开启型”实现：

- 未配置微信开放平台参数时，登录页不展示微信入口。
- 配置完整且启用后，远程服务模式登录页展示“微信扫码登录”。
- 微信登录成功后复用现有 `ard_token` cookie 和 `/api/auth/me` 登录态，不引入第二套会话体系。
- 本机模式不显示微信登录入口。
- 测试环境通过 mock 微信接口验证后端流程，不依赖真实微信开放平台。

## 必要配置

建议使用环境变量或服务端配置项：

```text
WECHAT_LOGIN_ENABLED=true
WECHAT_APP_ID=...
WECHAT_APP_SECRET=...
WECHAT_REDIRECT_URI=https://your-domain.example.com/api/auth/wechat/callback
WECHAT_AUTO_REGISTER=true
```

说明：

- `WECHAT_REDIRECT_URI` 必须与微信开放平台后台配置的授权回调域匹配。
- `WECHAT_APP_SECRET` 只能保存在服务端，不得下发到前端。
- `WECHAT_AUTO_REGISTER` 为 `true` 时，首次微信登录自动创建本系统用户目录；为 `false` 时，只允许已绑定用户登录。

## 拟定接口

后端：

- `GET /api/auth/wechat/login-url`
  - 返回微信授权 URL。
  - 未启用或配置不完整时返回明确错误，前端据此隐藏或置灰入口。

- `GET /api/auth/wechat/callback?code=...&state=...`
  - 校验 `state` 防止 CSRF。
  - 使用 `code` 换取微信用户身份。
  - 查找或创建本系统用户。
  - 调用现有 token 逻辑设置 `ard_token`。
  - 登录成功后重定向到 `/`。

前端：

- `auth.ts` 增加获取微信登录 URL 的方法。
- `Login.vue` 在远程模式且微信登录可用时展示微信入口。
- 微信登录完成后仍通过现有 `auth.init()` 刷新登录态。

## 用户身份映射

推荐在用户表中增加微信身份字段：

- `wechat_openid`
- `wechat_unionid`
- `wechat_nickname`
- `wechat_avatar_url`

优先使用 `unionid` 作为跨应用稳定身份；没有 `unionid` 时使用当前网站应用下的 `openid`。

自动创建用户名时建议使用稳定前缀，例如：

```text
wx_<openid_or_unionid_hash>
```

用户可见昵称只作为展示信息，不作为权限判断主键。

## 验证要求

实现时至少覆盖以下测试：

- 未启用微信登录时，接口返回不可用状态。
- 配置缺失时，不泄露 secret，错误信息可读。
- `state` 不匹配时拒绝 callback。
- 微信接口返回错误时拒绝登录。
- 首次微信登录可自动创建用户并初始化用户目录。
- 已存在微信用户再次登录复用原用户。
- 登录成功后设置 `ard_token` cookie，`/api/auth/me` 返回当前用户。

真实联调前，需要先确认：

- 微信开放平台网站应用已审核通过。
- 授权回调域已配置并可公网访问。
- 服务端部署地址使用 HTTPS。
- 回调路径能够到达当前 FastAPI 服务。
