# Excel 功能清单 → 全套交付物生成方案

## CLI

```bash
# 必要参数：指定源文件
python -m cosmic_tool.main --from-excel "功能清单.xlsx"

# 步骤参数（四选一，或 --gen-all 全流程）：
--gen-fpa        第1步：生成 FPA工作量评估.xlsx
--gen-cosmic     第2步：生成 项目功能点拆分表.xlsx（需第1步完成）
--gen-require    第3步：生成 项目需求清单.xlsx（需第2步完成）
--gen-docx       可选：生成 项目需求说明书.docx（无依赖，可随时执行）
--gen-all        全流程：按依赖顺序自动执行

# 示例
python -m cosmic_tool.main --from-excel "功能清单.xlsx" --gen-fpa
python -m cosmic_tool.main --from-excel "功能清单.xlsx" --gen-cosmic
python -m cosmic_tool.main --from-excel "功能清单.xlsx" --gen-require
python -m cosmic_tool.main --from-excel "功能清单.xlsx" --gen-all
```

可选 `--output-dir` 指定输出目录，默认输入文件所在目录。

### --gen-all 自动处理流程

`--gen-all` 全自动执行时，核减后工作量无法自动求和（需用户逐行填写），因此采用以下策略：

```
① 生成 FPA工作量评估.xlsx（核减后工作量默认 = FPA工作量）
   → 系统读取 FPA.xlsx，对核减后工作量列求和
   → 回填到 功能清单.xlsx → sheet 5.FPA核减后的工作量
    ↓
② 生成 项目功能点拆分表.xlsx
   CFP 上限 = FPA核减后的工作量 × CFP数量限制倍数
   → 系统读取 CFP 总和
    ↓
③ 生成 项目需求清单.xlsx
   送审功能点 = CFP 总和
```

> 如需精确的核减后工作量，建议分步执行：
> `--gen-fpa` → 在 FPA.xlsx 中逐行填写核减后工作量 → `--gen-cosmic` → `--gen-require`

## 源文件

`功能清单.xlsx` 含8个 sheet：

| Sheet | 内容 | 用途 |
|-------|------|------|
| `模板说明` | 说明文字 | 不参与处理 |
| `1、工单需求内容录入` | 工单信息 + 总体描述/建设目标/建设必要性/系统概况 | 项目信息 + docx Section 1-2 文本 |

| `4、项目需求说明书元数据录入` | docx 文件名/标题/日期/应用类型/可靠性描述等 | docx 段落替换 |
| `5、预估工作量和FPA核减后的工作量元数据录入` | 预估工作量、FPA核减工作量、CFP限制 | CFP 上限 + 送审数据 |

## 中间文件

解析 `功能清单.xlsx` 后生成两个 Markdown 中间文件：

| 中间文件 | 包含数据 | 数据来源 |
|---------|---------|---------|
| `功能清单模块树.md` | 模块树（入口/一级/二级/三级/客户端类型/三级模块整体功能描述/功能过程/过程类型/过程描述） | `2、功能清单内容录入` |

## 占位符替换与 AI 生成标记约定

`文档元数据.md` 中的值可能包含 `【占位符】`，占位符对应的实际值在同一 Excel 文件中。

### 占位符替换规则

| 占位符 | 替换为 | 来源 |
|--------|--------|------|
| `【工单编号】` | 工单编号的值 | `1、工单需求内容录入.工单编号` |
| `【工单标题】` | 工单标题的值 | `1、工单需求内容录入.工单标题` |
| `【工单内容】` | 工单内容的值 | `1、工单需求内容录入.工单内容` |
| `【子系统（模块）】` | 子系统名称 | `3、FPA工作量评估元数据录入.子系统（模块）` |
| `【三级模块整体功能描述】` | 各三级模块的整体功能描述 | `2、功能清单内容录入` |
| `【功能过程描述】` | 各功能过程的描述 | `2、功能清单内容录入` |
| `【三级模块】` | 各三级模块的名称 | `2、功能清单内容录入` |

### AI 生成标记

值以 `#AI生成#` 开头时，表示该段文本需要调用 AI 生成。处理流程：

1. 先将值中的 `【占位符】` 替换为实际内容
2. 去掉 `#AI生成#` 前缀，剩余文本作为 AI 提示词
3. 调用 AI 生成最终文本

示例：

| 原始值 | 处理后提示词 |
|--------|-------------|
| `#AI生成#基于【工单内容】` | `基于为满足垂直行业场景化营销和对外合作赋能需求，拟构建...` |
| `#AI生成#基于【三级模块整体功能描述】` | `基于垂直行业管理，支持搜索、添加、编辑、删除行业` |
| `【工单标题】` | `关于构建垂直行业场景化营销的需求`（直接替换，不调 AI） |

## FPA核减后的工作量取值逻辑

`5、预估工作量和FPA核减后的工作量元数据录入.FPA核减后的工作量（人/天）` 有三种取值方式：

| 方式 | 说明 |
|------|------|
| 手动输入 | 用户直接在单元格中输入数值 |
| 系统自动求和 | 系统运行时，对 `FPA工作量评估.xlsx` 中核减后工作量列自动求和，回填到此单元格 |
| 界面手动输入 | 用户在系统运行时的界面中手动输入 |

优先级：自动求和 > 界面手动输入 > 单元格原值。

## 输出生成顺序（依赖关系）

三个 Excel 输出有严格的依赖关系，必须按序生成：

```
第一步：FPA工作量评估.xlsx
        生成后确定 FPA核减后的工作量 的值（手动/自动求和/界面输入）
             ↓
第二步：项目功能点拆分表.xlsx
        CFP 总和不得超过 FPA核减后的工作量 × CFP数量限制倍数
             ↓
第三步：项目需求清单.xlsx
        送审功能点 = CFP 总和（来自第二步）
```

### 完整流水线

```
功能清单.xlsx
  │
  ├→ excel_source.py 解析
  │     ① 2、功能清单内容录入 → 功能清单模块树.md
  │     ② 其余 sheet → 文档元数据.md
  │
  ├→ gen_docx.py                            （可与第1步并行）
  │     输入: 功能清单模块树.md + 文档元数据.md
  │     处理: 替换占位符 → 对 #AI生成# 标记调 AI → 替换 docx 模板段落
  │     模板: 项目需求说明书.docx
  │     → 输出 项目需求说明书.docx
  │
  ├→ [第1步] gen_xlsx.py — FPA
  │     输入: 功能清单模块树.md + 文档元数据.md
  │     处理: init_fpa_template_md → ai_fill_fpa_data_md → write_to_template
  │     模板: FPA工作量评估.xlsx
  │     → 输出 FPA工作量评估.xlsx
  │     → 确定 FPA核减后的工作量 的值（自动求和/用户输入/单元格值）
  │
  ├→ [第2步] 功能清单模块树.md → 复用现有 COSMIC 链路
  │     输入: 功能清单模块树.md + FPA核减后的工作量
  │     处理: init_base_data_md → ai_fill_cosmic_data_md
  │           约束: CFP 总和 ≤ FPA核减后的工作量 × CFP数量限制倍数
  │           → parse_md_to_items → write_to_template
  │     模板: data/excel_template.xlsx
  │     → 输出 项目功能点拆分表.xlsx
  │     → 确定 CFP 总和
  │
  └→ [第3步] gen_xlsx.py — 需求清单
       输入: 功能清单模块树.md + 文档元数据.md + CFP 总和
       模板: 项目需求清单.xlsx
       → 输出 项目需求清单.xlsx
```

## 输出2：功能点拆分表.xlsx — 第2步生成

将 `功能清单模块树.md` 送入现有 MD 工作流，受 FPA 核减后工作量约束。

### 流程

```
功能清单模块树.md
  → init_base_data_md()    生成拆分表模板 MD（≈ --init-md）
  → ai_fill_cosmic_data_md()   AI 填充 COSMIC 数据（≈ --fill-md）
       约束: CFP 总和 ≤ FPA核减后的工作量 × CFP数量限制倍数
       ↓
      AI 填充完成后，对 CFP 列求和，将 CFP 总量写回 MD 文件元数据区
  → parse_md_to_items() → write_to_template()
      模板: data/excel_template.xlsx
      → 输出 项目功能点拆分表.xlsx
```

### 配置来源

| 配置项 | 来源 |
|--------|------|
| 客户需求 | `文档元数据.md.6.客户需求`（= 工单标题） |
| 功能用户-发起者 | `文档元数据.md.6.功能用户-发起者` |
| 功能用户-接收者 | 按 `文档元数据.md.6.功能用户-接收者判定` 规则，根据客户端类型判定 |
| 触发事件 | `文档元数据.md.6.触发事件` |
| CFP 计算公式 | `文档元数据.md.6.CFP计算公式` |
| CFP 上限 | FPA核减后的工作量 × CFP数量限制倍数 |

### 功能用户接收者判定

`ai_fill_cosmic_data_md()` 阶段根据客户端类型判定接收者：

| 条件 | 接收者 |
|------|--------|
| 客户端类型含"后台" | 后台管理员 |
| 客户端类型含"前台" | 普通用户 |
| 客户端类型含"渠道" | 渠道人员 |
| 以上都不含 | 操作员 |

### CFP 上限约束

CFP 总和不得超过 `FPA核减后的工作量 × CFP数量限制倍数`：
1. `ai_fill_cosmic_data_md()` 时将上限值传入 prompt
2. AI 返回后校验 CFP 总和，若超出则按比例等额缩放至限制值以内

### 跨输出共享字段

`1、环境图` sheet 中的建设目标、建设必要性，与 docx 的 1.2建设目标、1.3建设必要性同源（`文档元数据.md.1`），docx 和 xlsx 各自读取。

## 输出4：FPA工作量评估.xlsx — 第1步生成

参照输出2 的模式，分两步处理：规则生成骨架 → AI 填充描述。

### 流程

```
功能清单模块树.md + 文档元数据.md
  → init_fpa_template_md()    生成 FPA 模板 MD（规则骨架）
  → ai_fill_fpa_data_md()     AI 填充 F/G 列
       ↓
      AI 完成后对各列求和，写回 MD 文件元数据区
  → parse_fpa_to_items() → write_to_template()
      模板: FPA工作量评估.xlsx
      → 输出 FPA工作量评估.xlsx
  → 确定 FPA核减后的工作量（核减后工作量列求和 / 用户输入）
```

### 第一步：规则生成骨架（init_fpa_template_md）

每条 `功能过程` 展开为 2 行（界面行 + 接口行），结构化字段由规则填充：

| 列 | 界面行 | 接口行 |
|----|--------|--------|
| E 类型 | `EI` | `ILF` |
| D 功能点名称 | `【{客户端类型}】{一级}-{二级}-{三级}-{功能过程}-界面开发` | `【{客户端类型}】{一级}-{二级}-{三级}-{功能过程}-接口开发` |
| F 计算依据归类 | **留空（由 AI 填充）** | **留空（由 AI 填充）** |
| G 计算依据说明 | **留空（由 AI 填充）** | **留空（由 AI 填充）** |
| H 变更状态 | `功能过程类型` | `功能过程类型` |
| J 调整值 | 2 | 1 |
| K 要素数量 | 1 | 1 |

其余字段由规则直接填入：
- B 子系统 → `文档元数据.md.3.子系统（模块）`
- C 资产标识 → `文档元数据.md.3.资产标识`
- I 基准值 → 公式 `文档元数据.md.3.基准值公式`
- L FPA工作量 → 公式 `文档元数据.md.3.FPA工作量公式`

### 第二步：AI 填充（ai_fill_fpa_data_md）

AI 读取 `功能过程描述` + `E 类型` + `文档元数据.md.3.计算依据归类判定原则`，同时填充 F 列和 G 列：

- **F 列（计算依据归类）**：AI 根据功能过程描述，从判定原则中选出最匹配的一项
- **G 列（计算依据说明）**：AI 参考功能过程描述，展开为详细的界面/接口说明

例如功能过程描述为：

> 点击添加垂直行业按钮跳转至添加界面，要素包含垂直行业名称、确认按钮、取消按钮

AI 输出界面行：

| 列 | 值 |
|----|-----|
| F 计算依据归类 | `EI:1)修改或增加界面的个数` |
| G 计算依据说明 | `【地市后台】垂直行业管理-垂直行业管理-添加垂直行业-界面开发，具体如下：\n1、新增添加垂直行业界面，要素包含垂直行业名称、确认按钮、取消按钮` |

## 目录处理

Word 的目录（TOC）是一个字段代码，内容变更后需要 Word 重新计算页号，程序无法自动刷新。
生成后在文档开头加一句提示：`（打开后右键目录更新域即可刷新页号）`。
