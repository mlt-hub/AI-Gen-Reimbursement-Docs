# FPA 人工审阅反馈闭环方案

## 背景

当前 `gen-fpa` 已支持正式 Excel、`FPA工作量评估-check.xlsx` 和 Web 预览审核信息。后续需要增加一条人工审阅反馈闭环：

```text
HTML 审阅与校验
  -> 导出 FPA人工审阅.xlsx
  -> 人工填写审阅结果、审阅备注和修订字段
  -> 回填生成功能清单-录入文档-已审阅.xlsx
  -> 下次 gen-fpa 读取审阅反馈并增强生成
```

目标不是让人工审阅结果覆盖原始功能清单，而是把人工确认、修订、删除和备注转化为下一次 FPA 生成的稳定约束。

## 文件定位

| 文件 | 用途 |
|---|---|
| 正式 Excel | 最终交付、归档 |
| `FPA工作量评估-check.xlsx` | 系统校验、排查、内部核对 |
| `FPA人工审阅.xlsx` | 面向人工确认、批注、流转 |
| `功能清单-录入文档-已审阅.xlsx` | 带人工审阅反馈的功能清单副本 |

`功能清单-录入文档-已审阅.xlsx` 应保留原功能清单 Sheet 不变，并新增 `FPA人工审阅` Sheet。原功能清单继续作为源输入，`FPA人工审阅` 作为增强上下文。

## 审阅 Sheet 字段

`FPA人工审阅` Sheet 推荐字段：

```text
#
三级模块
功能过程
新增/修改功能点
类型
生成方式
计算依据归类
计算依据说明
审阅结果
审阅备注
修订后新增/修改功能点
修订后类型
```

为了支持稳定回填和下次匹配，应保留隐藏字段：

```text
source_row_index
module_key
fpa_row_id
source_process_hash
```

| 字段 | 说明 |
|---|---|
| `source_row_index` | 对应源功能清单行号 |
| `module_key` | 三级模块上下文标识 |
| `fpa_row_id` | FPA 生成结果行唯一标识 |
| `source_process_hash` | 源功能过程摘要 hash，用于判断审阅反馈是否过期 |

## 审阅结果

`审阅结果` 使用固定下拉值：

```text
通过
需修改
删除
待确认
```

| 审阅结果 | 下次 gen-fpa 行为 |
|---|---|
| 通过 | 优先沿用该行，不让 AI 随意改写 |
| 需修改 | 有修订字段时采用修订字段；只有审阅备注时交给 AI 按备注重新生成 |
| 删除 | 默认排除该功能点，并把删除原因作为负向约束 |
| 待确认 | 不强制采用，但把审阅备注作为 AI 参考 |

## 下次 gen-fpa 优先级

下次 `gen-fpa` 读取 `功能清单-录入文档-已审阅.xlsx` 时，推荐优先级为：

```text
人工修订
  > 人工通过
  > 删除排除
  > 审阅备注约束 AI
  > 常规 AI 生成
  > 规则兜底
```

### 人工修订

当 `审阅结果 = 需修改`，且存在 `修订后新增/修改功能点` 或 `修订后类型` 时：

```text
优先使用修订后新增/修改功能点。
优先使用修订后类型。
缺失的字段可沿用原生成结果或由规则补齐。
```

人工修订结果应进入正式 FPA 行，并保留 `生成方式` 或额外内部标记，便于 check Excel 展示来源。

### 人工通过

当 `审阅结果 = 通过` 时：

```text
优先沿用原新增/修改功能点、类型、计算依据归类、计算依据说明。
除非源功能过程发生明显变化，否则不再让 AI 改写该行。
```

### 删除排除

当 `审阅结果 = 删除` 时：

```text
下次生成默认不再输出该新增/修改功能点。
审阅备注作为删除原因，进入 AI 负向约束。
```

如果源功能过程发生变化，删除意见只作为参考，不强制排除。

### 审阅备注约束 AI

第二版增强重点是把 `审阅备注` 注入 AI prompt。适用场景：

```text
审阅结果 = 需修改，但未填写修订后字段。
审阅结果 = 待确认，且填写了审阅备注。
审阅结果 = 删除，需要提醒 AI 不要重复生成相同误判。
```

Prompt 中推荐增加结构化片段：

```text
人工审阅反馈：
- 原新增/修改功能点：...
- 原类型：...
- 审阅结果：需修改
- 审阅备注：应拆分为查询和导出两个功能点。
- 修订后新增/修改功能点：...
- 修订后类型：...
```

AI 必须优先遵循人工审阅反馈；当反馈与当前源功能清单明显冲突时，应以当前源功能清单为准，并在 `计算依据说明` 中说明处理理由。

## 反馈匹配

优先匹配：

```text
fpa_row_id
```

兜底匹配：

```text
module_key + 新增/修改功能点 + 类型
module_key + source_row_index
module_key + 功能过程
```

匹配失败时，不应阻断生成。系统可把该条反馈写入 warning，供 `FPA工作量评估-check.xlsx` 或 Web 审核面板展示。

## 过期保护

审阅反馈来自旧版本功能清单时可能已经过期。推荐使用 `source_process_hash` 判断：

```text
hash 未变化：
  审阅反馈可强约束使用。

hash 已变化：
  通过、需修改、删除都降级为 AI 参考。
  不强制沿用或排除。
  记录 warning，提示源功能过程已变化。
```

这样可以避免用户更新功能清单后，旧的删除或修订意见错误影响新需求。

## 对 check Excel 和 Web 审核面板的影响

后续应在审核信息中展示人工反馈来源：

```text
生成方式：人工通过 / 人工修订 / AI / 规则 / 兜底
Warnings：审阅反馈过期、审阅反馈匹配失败、删除排除生效
规则命中详情：人工审阅反馈是否采用
AI原始返回：包含传入 AI 的人工审阅反馈摘要
```

用户可见字段仍遵循 FPA 审阅术语规范：

```text
新增/修改功能点
类型
生成方式
计算依据归类
计算依据说明
```

## 实施切片

### 第一切片：读取审阅反馈

目标：

```text
识别功能清单-录入文档-已审阅.xlsx 中的 FPA人工审阅 Sheet。
解析审阅结果、审阅备注和修订字段。
无 FPA人工审阅 Sheet 时，现有 gen-fpa 行为不变。
```

验收：

```text
通过 / 需修改 / 删除 / 待确认 均能被解析。
隐藏映射字段缺失时有兜底匹配策略。
格式错误只产生可理解错误或 warning，不静默失败。
```

### 第二切片：应用强约束

目标：

```text
通过：沿用原行。
需修改且有修订字段：使用修订字段。
删除：排除原功能点。
```

验收：

```text
已通过行下次生成保持稳定。
需修改行采用修订后新增/修改功能点和修订后类型。
删除行不再出现在正式 FPA 行中。
源功能过程 hash 变化时，强约束降级为参考。
```

### 第三切片：审阅备注进入 AI prompt

目标：

```text
把待确认、需修改、删除的审阅备注作为结构化上下文传给 AI。
AI cache key 纳入审阅反馈摘要，避免复用旧缓存。
```

验收：

```text
只有审阅备注、没有修订字段的需修改行会进入 AI prompt。
待确认备注会进入 AI prompt，但不强制采用。
删除备注会作为负向约束进入 AI prompt。
审阅反馈变化会触发新的 AI cache key。
```

### 第四切片：审核展示

目标：

```text
check Excel 和 Web 审核面板展示人工反馈是否被采用。
反馈过期、匹配失败、删除排除等情况进入 warning。
```

验收：

```text
用户能看到某行来自人工通过、人工修订、AI 或规则。
Warnings 能定位到对应审阅反馈。
AI原始返回或 prompt 调试信息能看到人工审阅反馈摘要。
```

## 非目标

第一阶段不做：

```text
直接覆盖原始功能清单 Sheet。
把多条新增/修改功能点强塞回同一个功能过程单元格。
把审阅备注当作唯一事实来源。
无条件复用旧审阅意见。
```

## 风险

| 风险 | 控制方式 |
|---|---|
| `fpa_row_id` 不稳定 | 使用 `module_key + 新增/修改功能点 + 类型` 兜底 |
| 用户修改源功能清单后旧意见误用 | 使用 `source_process_hash` 过期保护 |
| AI 忽略审阅备注 | 在 prompt 中结构化传入，并增加 golden case |
| 删除意见误伤新需求 | hash 变化时删除只作为参考 |
| 文件太多用户分不清 | 文件名和 Sheet 名明确区分正式、check、人工审阅、已审阅功能清单 |
