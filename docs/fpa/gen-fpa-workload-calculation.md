# gen-fpa 阶段工作量计算逻辑

## 当前口径

在 `gen-fpa` 阶段，当前工作量口径很直接：

```text
单行 FPA 工作量 = 调整值 × 要素数量
总工作量 = 所有行工作量求和
```

这是一套简化估算模型。它没有展开 IFPUG 风格的低/中/高复杂度、DET、FTR、RET 等细分判断，而是把工作量映射到两项核心字段：

- `调整值`
- `要素数量`

## 字段来源

每一行 FPA 数据会包含以下关键字段：

- `新增/修改功能点`
- `类型`
- `计算依据归类`
- `计算依据说明`
- `变更状态`
- `调整值`
- `要素数量`

其中：

- `调整值` 根据 FPA `类型` 生成。
- `要素数量` 默认是 `1`，如果行数据中存在 `element_count`，则使用该值。

## 当前调整值规则

当前实现中，`调整值` 的规则为：

```text
EI => 2
其他类型 => 1
```

也就是说：

```text
类型为 EI 的功能点，默认工作量权重为 2
类型为 EO / EQ / ILF / EIF 等其他功能点，默认工作量权重为 1
```

## Markdown 汇总

生成 FPA Markdown 时，系统会根据每一行的：

```text
调整值 × 要素数量
```

计算总工作量，并写入 FPA 工作量总和文件。

## Excel 公式

导出 Excel 时，系统会将数据写入模板：

| 列 | 字段 |
|---|---|
| J 列 | 调整值 |
| K 列 | 要素数量 |
| L 列 | FPA 工作量 |

L 列公式为：

```text
=J3*K3
=J4*K4
...
```

汇总公式为：

```text
=SUM(L3:L最后一行)
```

## 结论

当前 `gen-fpa` 阶段的工作量计算，本质上是一个简化估算模型：

```text
FPA 工作量 = 调整值 × 要素数量
```

其中 `调整值` 主要由 `类型` 决定，`要素数量` 表示该类功能点的数量。
