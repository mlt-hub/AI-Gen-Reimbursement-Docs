# 生成结果审阅术语规范

## 目标

统一 AI/规则生成结果在审阅、修改、预览页面中的字段命名和展示结构，避免 FPA、COSMIC 等不同生成流程各自使用一套相近但不一致的术语。

本规范面向“生成结果需要人工审阅”的页面。用户看到的业务术语必须优先使用用户指定术语；工程内部可以使用稳定的抽象字段承载不同业务域的展示文案。

---

## 用户指定规范术语

以下术语为用户制定的界面与文档规范，FPA 相关页面必须统一使用：

| 规范术语 | 使用场景 | 说明 |
|---|---|---|
| 新增/修改功能点 | 结果项名称 | 展示 AI 或规则生成、补充、调整后的功能点内容 |
| 类型 | 类型字段 | 展示 FPA 类型或其他业务域中的类型/分类结果 |
| 计算依据归类 | 归类依据 | 展示为什么被归到当前类型或类别 |
| 计算依据说明 | 详细说明 | 展示更完整的计算、规则、推理或上下文说明 |

不要在 FPA 界面中混用以下替代表述：

- 功能点类型
- 类型判定依据
- 功能点说明
- 归类
- 说明详情
- 依据说明

如需表达“生成来源”，使用 `生成方式`，例如规则、AI、兜底。

---

## 抽象字段的作用

抽象字段不是给用户看的术语，而是给系统设计、组件复用、数据模型和跨生成器文档使用的中间层。

它的作用是让 FPA、COSMIC、未来其他生成器都能共享同一种审阅结构，同时在界面上显示各自业务域的规范术语。

| 抽象字段 | FPA 显示术语 | COSMIC 可映射术语 | 用途 |
|---|---|---|---|
| 结果项 | 新增/修改功能点 | 新增/修改功能过程 | 用户需要审阅或修改的业务条目 |
| 类型 | 类型 | 数据移动类型 | 结果项所属类别、类型或计量口径 |
| 归类依据 | 计算依据归类 | 计算依据归类 | 当前类型/类别的判定依据 |
| 说明依据 | 计算依据说明 | 计算依据说明 | 更完整的计算、规则、推理和上下文说明 |
| 生成来源 | 生成方式 | 生成方式 | 结果来自规则、AI、兜底或混合流程 |

工程内部可以围绕这些抽象字段设计通用组件，例如 `ReviewResultTable`、`ReviewResultRow`。组件渲染时必须通过业务域映射显示用户指定术语。

---

## 通用审阅页面结构

生成结果审阅页面推荐遵循以下结构：

| 顺序 | 抽象字段 | FPA 表头 |
|---|---|---|
| 1 | 序号 | # |
| 2 | 结果项 | 新增/修改功能点 |
| 3 | 类型 | 类型 |
| 4 | 生成来源 | 生成方式 |
| 5 | 归类依据 | 计算依据归类 |

行级详情区域统一展示：

```text
计算依据说明
```

推荐把长说明放在行级详情中，不要把所有说明集中藏到页面底部的总折叠块里。这样用户审阅某一行时，可以在同一上下文中看到名称、类型、归类依据和详细说明。

---

## FPA 与 COSMIC 的复用规则

FPA 当前固定使用：

```text
新增/修改功能点 / 类型 / 生成方式 / 计算依据归类 / 计算依据说明
```

COSMIC 审阅页使用 COSMIC 业务对象命名，不直接套用 FPA 的“功能点”表达。除非 COSMIC 标准另有明确命名，`计算依据归类` 与 `计算依据说明` 应保持一致。

示例：

| 抽象字段 | FPA | COSMIC 示例 |
|---|---|---|
| 结果项 | 新增/修改功能点 | 新增/修改功能过程 |
| 类型 | 类型 | 数据移动类型 |
| 归类依据 | 计算依据归类 | 计算依据归类 |
| 说明依据 | 计算依据说明 | 计算依据说明 |

## COSMIC 审阅术语映射

COSMIC 相关预览、审阅、校验报告和人工确认页面应使用以下用户可见术语：

| 数据或字段 | 用户可见术语 | 说明 |
|---|---|---|
| `items[].process` | 新增/修改功能过程 | 人工审阅的核心业务对象。 |
| `items[].module_l1/module_l2/module_l3` | 模块路径 | 功能过程归属的一、二、三级模块。 |
| `items[].user` | 功能用户 | 触发或接收数据移动的功能用户。 |
| `items[].trigger` | 触发事件 | 启动功能过程的事件。 |
| `items[].movements[]` | 数据移动 | 功能过程内的 E/X/R/W 子过程。 |
| `movements[].move_type` | 数据移动类型 | 标准值为 `E`、`X`、`R`、`W`。 |
| `movements[].data_group` | 数据组 | 数据移动涉及的兴趣对象数据组。 |
| `movements[].data_attrs` | 数据属性 | 数据组中的属性清单。 |
| `movements[].reuse` | 复用度 | 展示新增、复用、利旧等口径。 |
| CFP 字段或公式结果 | CFP | COSMIC 功能规模数值。 |
| `status` | 校验状态 | 展示 `通过`、`待审`、`阻断`。 |
| `issues` / `review_items` | 审阅项 | 需要人工确认的结构化问题。 |
| `issues[].code` | 问题代码 | 稳定定位规则或生成问题。 |
| `issues[].field` | 字段位置 | 问题对应的数据字段。 |
| `issues[].message` | 问题说明 | 面向人工的简短说明。 |
| `issues[].details` | 依据 | 命中词、功能用户匹配、模块匹配或规则说明。 |
| `review_items[].review_id` | 审阅项 ID | 稳定锚点，可用于人工确认状态绑定；一般不作为主列展示。 |

COSMIC 页面不使用以下 FPA 或泛化同义词替代：

- 功能点类型
- 类型判定依据
- 功能点说明
- 说明详情
- 新增/修改功能点
- COSMIC 功能点
- 子功能点

COSMIC 审阅表格的主阅读顺序建议为：

| 顺序 | 表头 |
|---|---|
| 1 | # |
| 2 | 模块路径 |
| 3 | 新增/修改功能过程 |
| 4 | 功能用户 |
| 5 | 触发事件 |
| 6 | 数据移动 |
| 7 | 数据移动类型 |
| 8 | 复用度 |
| 9 | CFP |
| 10 | 校验状态 |
| 11 | 审阅项 |

行级详情区域统一展示：

```text
计算依据说明
依据
```

`计算依据说明` 用于 AI 或规则给出的业务解释；`依据` 用于结构化校验 issue 的命中词、匹配来源和规则说明。

---

## 验收标准

- FPA 页面和文档中统一使用用户指定的四个规范术语。
- 表格首要阅读对象为 `新增/修改功能点`，不要让 `类型` 抢在业务对象前。
- `计算依据归类` 用于短依据，`计算依据说明` 用于长说明。
- 抽象字段只出现在工程设计文档或代码结构中，不直接暴露给最终用户。
- COSMIC 审阅页使用 `新增/修改功能过程`、`数据移动`、`数据移动类型`、`功能用户`、`触发事件`、`审阅项`、`依据` 等术语，不混用 FPA 禁用同义词。
- 新增其他生成器审阅页时，先补充业务术语映射，再复用通用审阅结构。
