# 生成页控件精简待办

## 背景

生成页需要把真正的主流程收紧到同一视觉区域内。`操作模式`、`功能清单 .xlsx 路径（或项目目录）`、`开始生成` 应并排展示；`高级参数（FPA策略与运行参数）` 放在其下方。生成页不再展示单独的 `FPA预览` 按钮，但保留从 `预览 -> FPA预览` 进入的入口。

同时，生成页需要增加 `运行详情 / 排错信息` 次级入口，用于查看运行中和失败后的任务详情。新任务、重跑、定位参数、多任务查看、并发排队和刷新策略也要一并理清。

## 主流程

- `操作模式`、`功能清单 .xlsx 路径（或项目目录）`、`开始生成` 并排展示。
- `高级参数（FPA策略与运行参数）` 位于主操作区第一行下方。
- `开始生成` 是主按钮。
- `运行详情 / 排错信息` 是次级按钮。
- 生成页不展示 `FPA预览` 按钮。
- `预览 -> FPA预览` 入口保留。

## 页面动作定义

### 开始生成

- 每次点击都创建新的后端 session 和新的运行历史记录。
- 提交前清空当前页面的运行态、日志和阶段进展。
- 任务运行中时保持禁用或明确提示，不允许静默覆盖当前 session。
- 本机模式使用 `功能清单 .xlsx 路径（或项目目录）` 作为输入；目录模式继续按现有规则自动寻找 `.xlsx`。
- 远程模式使用当前选择的上传文件作为输入。
- 高级参数随新任务一起提交，并固化为本次任务的参数快照。

### 新任务

- `新任务` 不是后端任务创建入口，只是把生成页清空到准备下一次填写和生成的页面状态。
- `新任务` 只做前端页面清空：清空当前页面 session 状态、日志展示、阶段进展展示、交付物展示，并移除浏览器中的 `ard:lastSessionId`。
- `新任务` 不取消后端任务，不删除运行历史，不删除交付物文件，不删除后端 session 记录，也不创建新任务。
- `新任务` 建议只在任务完成、失败或已停止后显示；任务运行中只显示 `停止`。
- `新任务` 应保留用户已填或已选择的生成参数，包括操作模式、路径输入和高级参数。
- 点击 `新任务` 后，用户需要再次点击 `开始生成`，才会创建新的后端 session 和运行历史记录。

### 重新运行

- `重新运行` 属于 `运行详情 / 排错信息` 页面操作。
- 重新运行基于原任务参数快照创建新的 session，不复用原 session。

### 定位参数

- `定位参数` 只负责回到生成页并定位或高亮相关参数。
- `定位参数` 不自动创建新任务，也不隐式修改历史任务。

## 多任务与并发

### 多任务查看

- 任务列表层应支持同时查看多个任务的状态、来源、模式、输入、交付物和操作入口。
- 运行详情层应支持同时打开多个任务详情，优先用标签页或左右分栏承载，而不是在单页里无限展开多个任务日志。
- 每个任务详情都必须绑定具体 `session_id`，日志、阶段进展、生成过程和排错信息都按任务隔离。
- 多任务同时查看时，实时日志流优先只保留当前激活任务；其他任务可用轮询刷新或按需手动刷新。

### 并发控制

- 当任务数量达到系统上限时，新任务应进入排队状态，而不是直接失败。
- 排队时页面应明确提示用户当前排队位置、预计等待状态或“已进入等待队列”的提示文案。
- 用户应能取消排队中的新任务，但取消后不影响已在运行的其他任务。
- 每个任务必须使用独立输出目录，不能让多个任务写入同一个生成目录或同一个临时工作目录。
- 若用户重复点击 `开始生成`，前端应避免重复提交同一任务请求，改为显示当前任务排队或运行状态。
- 详情页应打开实时连接，持续接收当前任务的日志和阶段进展。
- 列表页只做轮询刷新，不应为每个任务都维持长连接。
- 当用户切换到某个任务详情时，才为该任务建立或恢复实时连接；离开详情页后可关闭实时连接或降级为轮询。

## 验收标准

- 用户进入生成页后，能在第一眼看到并完成模式选择、路径填写和开始生成。
- 桌面视口中主操作控件并排或紧凑排列，页面不再被单个控件纵向拉长。
- `高级参数（FPA策略与运行参数）` 位于主操作区第一行下方，和主生成流程保持同一区域。
- `运行详情 / 排错信息` 次级按钮在允许状态下可进入独立页面。
- 生成页不存在 `FPA预览` 按钮。
- `预览 -> FPA预览` 入口保留，FPA 预览能力仍可从预览区域进入。
- `生成过程` 区域展示阶段进展与产物，不使用下拉进度条。
- 运行详情页支持查看、复制日志、重新运行和定位参数。
- `开始生成` 创建新的 session 和运行历史记录，不复用旧 session。
- 新任务提交前会清空当前页面运行态、日志和阶段进展。
- 任务运行中不会被新的 `开始生成` 静默覆盖。
- `新任务` 按钮只清空页面视图，不影响后端任务、运行历史、交付物文件和后端 session 记录。
- `新任务` 后保留操作模式、路径输入和高级参数，用户再次点击 `开始生成` 才创建新的后端任务。
- 任务列表可同时展示多个任务，多个运行详情可并行打开，但每个详情都必须绑定独立 `session_id`。
- 多任务同时查看时，只有当前激活任务保持实时日志流，其余任务按需刷新。
- 任务达到并发上限时，新任务进入排队并提示用户，可取消排队任务。
- 每个任务都使用独立输出目录，不能共享同一个输出或临时目录。
- 详情页负责实时连接，列表页只做轮询刷新。
- `重新运行` 基于历史参数快照创建新 session；`定位参数` 不自动启动任务。
- 移除按钮后不影响生成任务创建、运行中状态展示和错误提示。
- 所有新增或受影响测试通过。

## 风险与注意事项

- 如果生成页 `FPA预览` 按钮当前承载了生成前校验或调试能力，实施时需要确认这些能力是否迁移到 `开始生成` 流程、预览页入口或仅作为开发调试入口保留。
- 移除生成页按钮时要避免删除 `预览 -> FPA预览` 入口和仍被其他页面复用的预览组件。
- 高级参数移入主操作区后，需要确认折叠、默认值和任务提交参数不发生隐式变化。
- 运行详情页需要明确失败后与运行中的展示边界，避免用户在任务未产生日志时看到空白页。
- `重新运行` 和 `定位参数` 需要约束作用域，避免误改历史任务或当前页面状态。
- FPA 相关用户可见文案需继续遵循 `docs/fpa/result-review-terminology.md`，不要引入禁用同义词。
