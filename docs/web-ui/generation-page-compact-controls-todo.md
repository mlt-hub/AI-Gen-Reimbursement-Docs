# 生成页控件精简待办

## 背景

生成页当前需要强化主流程入口，把用户真正需要先完成的输入和动作放在同一视觉区域内。用户期望 `操作模式`、`功能清单 .xlsx 路径（或项目目录）`、`开始生成` 按钮可以并排显示，减少纵向跳转和页面扫描成本。同时，生成页不再提供单独的 `FPA预览` 按钮，避免把生成前操作拆成额外入口；FPA 预览能力保留从 `预览 -> FPA预览` 进入。

## 目标行为

- 生成页主操作区并排展示 `操作模式`、`功能清单 .xlsx 路径（或项目目录）`、`开始生成`。
- 在桌面宽度下，三项保持同一行或同一紧凑操作带；在移动端宽度下可自然换行，但仍保持主流程顺序。
- `高级参数（FPA策略与运行参数）` 移到生成页主操作区，放在 `操作模式`、`功能清单 .xlsx 路径（或项目目录）`、`开始生成` 下方。
- `开始生成` 是该区域的唯一主按钮。
- 生成页不展示 `FPA预览` 按钮，不再作为生成页单独操作入口展示。
- 保留从 `预览 -> FPA预览` 进入 FPA 预览能力。

## 拟修改范围

- Web 前端生成页组件：调整主操作区布局、按钮展示和响应式样式。
- Web 前端生成页高级参数区域：将 `高级参数（FPA策略与运行参数）` 放到主操作区输入行下方。
- Web 前端任务提交逻辑：确认移除 `FPA预览` 入口后，不影响 `开始生成` 的参数传递。
- Web 前端预览入口：保留 `预览 -> FPA预览` 路径，不随生成页按钮调整删除。
- 相关测试：更新页面渲染、按钮可见性和提交流程断言。
- 如现有文档或截图说明仍提到生成页 `FPA预览` 按钮，实施时同步更新。

## 待办任务

1. 定位生成页入口组件和 `FPA预览` 按钮来源。
2. 梳理 `操作模式`、输入路径和 `开始生成` 当前状态绑定关系。
3. 梳理 `高级参数（FPA策略与运行参数）` 当前位置、折叠状态和任务参数绑定关系。
4. 将 `操作模式`、路径输入和 `开始生成` 收敛到同一主操作区第一行，并设置桌面并排、窄屏换行的响应式布局。
5. 将 `高级参数（FPA策略与运行参数）` 移到主操作区第二行，位于第一行下方。
6. 移除生成页 `FPA预览` 按钮及其仅供该按钮使用的事件处理。
7. 确认 `预览 -> FPA预览` 入口仍存在，且可以进入原 FPA 预览能力。
8. 确认 `开始生成` 保持现有校验逻辑：路径为空、模式未选、任务运行中等状态仍有明确反馈。
9. 更新或新增前端测试，覆盖：
   - `操作模式`、路径输入、`开始生成` 可同时展示。
   - `高级参数（FPA策略与运行参数）` 展示在主操作区输入行下方。
   - 生成页不展示 `FPA预览` 按钮。
   - `预览 -> FPA预览` 入口仍可见或可导航。
   - 点击 `开始生成` 仍按当前参数创建任务。
10. 运行相关测试和前端构建检查。
11. 人工检查桌面和移动端布局，确认控件不重叠、文字不溢出。

## 验收标准

- 用户进入生成页后，能在第一眼看到并完成模式选择、路径填写和开始生成。
- 桌面视口中主操作控件并排或紧凑排列，页面不再被单个控件纵向拉长。
- `高级参数（FPA策略与运行参数）` 位于主操作区第一行下方，和主生成流程保持同一区域。
- 生成页不存在 `FPA预览` 按钮。
- `预览 -> FPA预览` 入口保留，FPA 预览能力仍可从预览区域进入。
- 移除按钮后不影响生成任务创建、运行中状态展示和错误提示。
- 所有新增或受影响测试通过。

## 风险与注意事项

- 如果生成页 `FPA预览` 按钮当前承载了生成前校验或调试能力，实施时需要确认这些能力是否迁移到 `开始生成` 流程、预览页入口或仅作为开发调试入口保留。
- 移除生成页按钮时要避免删除 `预览 -> FPA预览` 入口和仍被其他页面复用的预览组件。
- 高级参数移入主操作区后，需要确认折叠、默认值和任务提交参数不发生隐式变化。
- FPA 相关用户可见文案需继续遵循 `docs/fpa/result-review-terminology.md`，不要引入禁用同义词。
