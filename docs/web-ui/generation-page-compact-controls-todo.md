# 生成页控件精简待办

## 背景

生成页当前需要强化主流程入口，把用户真正需要先完成的输入和动作放在同一视觉区域内。用户期望 `操作模式`、`功能清单 .xlsx 路径（或项目目录）`、`开始生成` 按钮可以并排显示，减少纵向跳转和页面扫描成本。同时，生成页不再提供单独的 `FPA预览` 按钮，避免把生成前操作拆成额外入口；FPA 预览能力保留从 `预览 -> FPA预览` 进入。另需增加一个 `运行详情 / 排错信息` 次级按钮，进入独立页面查看运行中和失败后的任务详情。

## 目标行为

- 生成页主操作区并排展示 `操作模式`、`功能清单 .xlsx 路径（或项目目录）`、`开始生成`。
- 在桌面宽度下，三项保持同一行或同一紧凑操作带；在移动端宽度下可自然换行，但仍保持主流程顺序。
- `高级参数（FPA策略与运行参数）` 移到生成页主操作区，放在 `操作模式`、`功能清单 .xlsx 路径（或项目目录）`、`开始生成` 下方。
- `开始生成` 是该区域的唯一主按钮。
- `运行详情 / 排错信息` 作为次级按钮保留在生成页，可在任务运行中和失败后查看。
- `开始生成` 永远创建新任务，不复用旧 session；重跑和定位参数不混入主按钮逻辑。
- 生成页不展示 `FPA预览` 按钮，不再作为生成页单独操作入口展示。
- 保留从 `预览 -> FPA预览` 进入 FPA 预览能力。

## 新任务逻辑

- `开始生成` 是生成页的新任务入口，每次点击都创建新的后端 session 和新的运行历史记录。
- 创建新任务前，前端应清空当前页面的运行态、日志和阶段进展，避免新旧任务信息混杂。
- 如当前已有任务正在运行，`开始生成` 应保持禁用或给出明确提示，不允许静默覆盖当前运行中的 session。
- 本机模式创建新任务时，使用 `功能清单 .xlsx 路径（或项目目录）` 作为输入；如果传入项目目录，后端继续按现有规则自动寻找符合规范的功能清单 `.xlsx`。
- 远程模式创建新任务时，使用当前选择的上传文件作为输入。
- 高级参数随新任务一起提交，并固化为本次任务的参数快照，包括 FPA 策略、FPA 口径、规则集、确认模式、模型和运行参数。
- `重新运行` 属于 `运行详情 / 排错信息` 页面操作，基于原任务参数快照创建新的 session，不复用原 session。
- `定位参数` 只负责回到生成页并定位或高亮相关参数，不自动创建新任务，也不隐式修改历史任务。

## 新任务按钮逻辑

- `新任务` 按钮不是后端任务创建入口，只是把生成页清空到准备下一次填写和生成的页面状态。
- `新任务` 只做前端页面清空：清空当前页面 session 状态、日志展示、阶段进展展示、交付物展示，并移除浏览器中的 `ard:lastSessionId`。
- `新任务` 不取消正在运行的后端任务，不删除运行历史，不删除交付物文件，不删除后端 session 记录，也不创建新任务。
- `新任务` 建议只在任务完成、失败或已停止后显示；任务运行中只显示 `停止`，避免用户误以为清空页面等于停止任务。
- `新任务` 应保留用户已填或已选择的生成参数，包括操作模式、路径输入和高级参数，方便用户微调后再次点击 `开始生成`。
- 点击 `新任务` 后，用户需要再次点击 `开始生成`，才会创建新的后端 session 和运行历史记录。

## 拟修改范围

- Web 前端生成页组件：调整主操作区布局、按钮展示和响应式样式。
- Web 前端生成页高级参数区域：将 `高级参数（FPA策略与运行参数）` 放到主操作区输入行下方。
- Web 前端生成页次级入口：新增 `运行详情 / 排错信息` 按钮，进入独立详情页面。
- Web 前端任务提交逻辑：确认移除 `FPA预览` 入口后，不影响 `开始生成` 的参数传递。
- Web 前端新任务逻辑：明确 `开始生成` 创建新 session，并在提交前清空当前页面运行态。
- Web 前端新任务按钮逻辑：明确 `新任务` 仅清空页面视图，不触碰后端任务、历史和交付物。
- Web 前端预览入口：保留 `预览 -> FPA预览` 路径，不随生成页按钮调整删除。
- Web 前端运行详情页：支持查看、复制日志、重新运行、定位参数，并展示 `生成过程`。
- 相关测试：更新页面渲染、按钮可见性和提交流程断言。
- 如现有文档或截图说明仍提到生成页 `FPA预览` 按钮，实施时同步更新。

## 验收标准

- 用户进入生成页后，能在第一眼看到并完成模式选择、路径填写和开始生成。
- 桌面视口中主操作控件并排或紧凑排列，页面不再被单个控件纵向拉长。
- `高级参数（FPA策略与运行参数）` 位于主操作区第一行下方，和主生成流程保持同一区域。
- `运行详情 / 排错信息` 次级按钮在允许状态下可进入独立页面。
- 生成页不存在 `FPA预览` 按钮。
- `预览 -> FPA预览` 入口保留，FPA 预览能力仍可从预览区域进入。
- `生成过程` 区域展示阶段进展与产物，不使用下拉进度条。
- 运行详情页支持查看、复制日志、重新运行和定位参数。
- `开始生成` 创建新的 session 和运行历史记录，不复用旧 session。
- 新任务提交前会清空当前页面运行态、日志和阶段进展。
- 任务运行中不会被新的 `开始生成` 静默覆盖。
- `新任务` 按钮只清空页面视图，不影响后端任务、运行历史、交付物文件和后端 session 记录。
- `新任务` 后保留操作模式、路径输入和高级参数，用户再次点击 `开始生成` 才创建新的后端任务。
- `重新运行` 基于历史参数快照创建新 session；`定位参数` 不自动启动任务。
- 移除按钮后不影响生成任务创建、运行中状态展示和错误提示。
- 所有新增或受影响测试通过。

## 风险与注意事项

- 如果生成页 `FPA预览` 按钮当前承载了生成前校验或调试能力，实施时需要确认这些能力是否迁移到 `开始生成` 流程、预览页入口或仅作为开发调试入口保留。
- 移除生成页按钮时要避免删除 `预览 -> FPA预览` 入口和仍被其他页面复用的预览组件。
- 高级参数移入主操作区后，需要确认折叠、默认值和任务提交参数不发生隐式变化。
- 运行详情页需要明确失败后与运行中的展示边界，避免用户在任务未产生日志时看到空白页。
- `重新运行` 和 `定位参数` 需要约束作用域，避免误改历史任务或当前页面状态。
- FPA 相关用户可见文案需继续遵循 `docs/fpa/result-review-terminology.md`，不要引入禁用同义词。
