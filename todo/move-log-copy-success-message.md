# 调整运行日志复制成功提示位置

## 状态

- 状态：已实施
- 实施提交：`9c25206 fix: move log copy success notice`
- 实施文件：`web_app/src/views/TaskDetail.vue`
- 验证命令：`npm run build`
- 验证结果：通过，`vue-tsc -b && vite build` 成功完成。

## 背景

在任务运行详情页中，点击“复制日志”后，当前“日志已复制”成功提示显示在上方“运行详情”卡片区域内。

从页面信息层级看，该反馈由“复制日志”按钮触发，语义上属于“运行日志 / 日志与错误详情”区域。提示放在运行详情卡片中会让用户误以为该反馈与任务基础信息相关，也会拉开运行详情和运行日志之间的视觉距离。

## 目标行为

- 点击“复制日志”后，“日志已复制”提示显示在“运行日志”卡片上方。
- 提示应靠近“日志与错误详情”和“复制日志”按钮所在区域。
- 保留现有成功态样式、提示文案和自动消失逻辑。
- 不改变日志复制功能本身，不改变任务重新运行、定位参数、返回任务列表等操作。

## 实际修改范围

本次实际修改集中在任务运行详情页组件：

- 在 `web_app/src/views/TaskDetail.vue` 中新增 `logNotice`，专门承载“复制日志”成功反馈。
- 将“日志已复制”的渲染位置放到“运行日志 / 日志与错误详情”标题区下方、日志内容上方。
- 保留原有 `notice` 作为任务级提示，用于“任务已恢复到任务列表”等运行详情区域反馈。
- 在重新加载详情、重新运行、恢复任务时清空 `logNotice`，避免旧的日志复制提示残留。

未修改后端接口、日志生成逻辑、日志复制内容和交付物操作。

## 实施步骤

1. 定位任务运行详情页组件。
2. 找到“复制日志”按钮和“日志已复制”提示的状态及渲染位置。
3. 将成功提示的渲染位置从“运行详情”卡片区域移到“运行日志”卡片标题区域上方。
4. 保持提示样式与当前页面一致，必要时微调上下间距。
5. 确认提示不会遮挡“运行日志”标题、“复制日志”按钮或日志内容。
6. 如已有前端测试覆盖复制日志交互，更新断言；如无测试，至少进行手动验证。

## 风险与注意事项

- 如果复制提示状态被多个按钮复用，移动渲染位置前需要确认不会影响其他复制反馈。
- 如果运行日志为空或加载失败，也应确认提示区域不会造成布局异常。
- 如果页面使用统一 Alert 组件，优先复用现有组件，不新增重复样式。
- 本调整属于展示层改动，不应改动后端接口和日志生成逻辑。
