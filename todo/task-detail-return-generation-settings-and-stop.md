# 运行详情页操作命名与停止按钮改进计划

## 背景

运行详情页顶部当前提供 `返回任务列表`、`定位参数`、`重新运行` 等操作。

本轮讨论确认：

- `定位参数` 的实际行为不是恢复、应用或修改历史任务参数，而是回到生成页，并通过 URL query 定位或高亮某个生成设置控件。
- 顶部 `定位参数` 当前固定跳转到生成页的输入路径区域。
- 参数快照区域已有更细的跳转入口，可定位到操作模式、输入路径、高级参数、FPA 方案、FPA 策略、FPA 规则集、FPA 生成模式等设置项。
- 与直接点击工作台下的 `生成` 不同，详情页入口会携带 `focus` 和 `fromSession` 上下文，用于滚动和高亮相关设置项。
- 运行详情页已经有 `重新运行` 按钮，但运行中或排队中的任务缺少 `停止任务` 按钮；后端已有 `/api/cancel/{session_id}` 能力，生成页和任务列表也已使用该接口。

## 目标行为

1. 将运行详情页顶部按钮文案从 `定位参数` 改为 `返回生成设置`。
2. 保持现有跳转行为不变：点击后进入生成页，并定位到输入路径设置区域。
3. 保持参数快照区域的细粒度定位入口不变，继续显示具体设置项名称。
4. 在运行详情页为 `queued` 和 `running` 状态补充 `停止任务` 操作。
5. `停止任务` 成功后刷新运行详情页状态、历史记录和日志展示。
6. 任务处于 `done`、`error`、`cancelled` 等终态时不显示停止按钮，继续保留 `重新运行`。

## 命名说明

### 为什么不用 `定位参数`

`定位参数` 偏技术表达，用户不容易立即判断它是跳转、恢复、套用还是修改参数。

实际功能只是回到生成页并高亮设置项，因此应避免使用容易让人误解为“对历史参数执行操作”的名称。

### 为什么选择 `返回生成设置`

`返回生成设置` 同时表达了两个关键信息：

- `返回`：从运行详情页回到生成页。
- `生成设置`：目标是生成页中的配置控件，而不是重新运行任务或恢复历史参数。

相比 `返回生成任务`，`返回生成设置` 更贴近当前实际行为。`返回生成任务` 容易让用户预期回到任务现场，包括进度、日志或停止操作；而当前入口主要服务于查看或调整生成参数。

## 当前实现线索

- 运行详情页顶部按钮在 `web_app/src/views/TaskDetail.vue`，当前文案为 `定位参数`，跳转目标为 `/?focus=input&fromSession=<sessionId>`。
- 参数快照区域的细粒度入口同在 `TaskDetail.vue`，由 `focusTargets` 列表驱动。
- 生成页通过 `web_app/src/views/Home.vue` 中的 `focusGenerationField()` 读取 `route.query.focus`，映射到对应 DOM 后滚动并短暂高亮。
- 后端停止接口为 `POST /api/cancel/{session_id}`，位于 `web_app/routes/tasks.py`。
- 生成页已有取消逻辑可参考 `web_app/src/views/Home.vue`。
- 任务列表已有取消排队逻辑可参考 `web_app/src/views/Tasks.vue`。

## 拟修改文件范围

### 前端

- `web_app/src/views/TaskDetail.vue`
  - 将顶部 `定位参数` 按钮文案改为 `返回生成设置`。
  - 增加 `canCancel` 计算属性，覆盖 `queued` 和 `running` 状态。
  - 增加 `cancelTask()` 方法，调用 `/api/cancel/{sessionId}`。
  - 停止成功后关闭实时日志流，并刷新 session 状态、历史记录和日志。
  - 在顶部操作区按状态展示 `停止任务` 按钮。

### 测试

视现有前端测试能力补充其一：

- 若已有组件测试：补充运行详情页在 `queued` / `running` 下显示停止按钮、终态不显示。
- 若暂无合适前端组件测试：补充后端或端到端烟测说明，人工验证运行详情页停止行为。

## 实施步骤

1. 修改 `TaskDetail.vue` 顶部按钮文案：`定位参数` -> `返回生成设置`。
2. 在 `TaskDetail.vue` 中新增 `canCancel`：
   - `effectiveRunState === 'queued' || effectiveRunState === 'running'`
   - 需要确保无 sessionId 时不可用。
3. 在顶部按钮区新增 `停止任务`：
   - `v-if="canCancel"`
   - 使用危险或强调样式，保持与生成页现有停止按钮风格一致。
   - 点击时禁用按钮，避免重复提交。
4. 新增 `cancelTask()`：
   - 清理 `error`、`notice`、`logNotice`。
   - 调用 `apiFetch('/api/cancel/' + sessionId, { method: 'POST' })`。
   - 成功后关闭 event stream。
   - 重新加载 `sessionStatus`、`historyItem`、`logs`。
5. 检查 `queued`、`running`、`done`、`error`、`cancelled` 五类状态下按钮展示是否符合预期。
6. 更新或补充测试。

## 验收标准

- 运行详情页顶部不再出现 `定位参数` 文案，改为 `返回生成设置`。
- 点击 `返回生成设置` 后仍跳转到生成页，并定位或高亮输入路径区域。
- 参数快照区域的具体入口仍能分别定位到对应设置项。
- 排队中的任务详情页显示 `停止任务`，点击后任务状态变为 `已停止`。
- 运行中的任务详情页显示 `停止任务`，点击后任务进入停止流程，最终展示 `已停止`。
- 已完成、失败、已停止的任务详情页不显示 `停止任务`。
- `重新运行` 仍只在可重跑的终态 Web 任务上可用。
- 直接点击工作台下的 `生成` 不受影响，不携带详情页的 `focus` 上下文。

### 人工验证

1. 启动 Web 应用。
2. 创建一个排队或运行中的任务。
3. 打开该任务运行详情页。
4. 确认顶部按钮显示 `返回生成设置` 和 `停止任务`。
5. 点击 `返回生成设置`，确认生成页滚动并高亮输入路径。
6. 返回运行详情页，点击 `停止任务`，确认状态刷新为 `已停止`。
7. 对完成、失败、已停止任务确认不显示 `停止任务`，但符合条件时显示 `重新运行`。

## 风险与注意事项

- `返回生成设置` 只是跳转和高亮，不应恢复或覆盖当前生成页表单值。
- `fromSession` 当前主要用于上下文标识，不应在本次改名中引入额外副作用。
- 停止运行中任务不是立即杀死当前 API 调用；系统会在可检查取消点停止，需保持现有后端语义。
- 排队任务取消和运行中任务取消共用 `/api/cancel/{session_id}`，前端只需按状态刷新展示，不应自行推断后端内部状态。
