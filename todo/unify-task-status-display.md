# 统一任务详情页与生成页的状态展示

## 状态

- 状态：待实施
- 来源：任务详情页状态展示与生成页不一致
- 目标页面：
  - 生成页：`web_app/src/views/Home.vue`
  - 查看任务详情页：`web_app/src/views/TaskDetail.vue`
- 相关组件：
  - `web_app/src/components/GenerationProgress.vue`
  - `web_app/src/stores/steps.ts`

## 背景

当前生成页顶部“生成进度”区域会展示完整状态，例如：

- `已完成 · 生成 FPA`
- 辅助说明：`FPA 工作量评估已生成`

但查看任务详情页的基础信息卡片中，状态只显示为：

- `完成`

这会造成同一个任务在两个页面上的状态语义不一致。用户从生成页进入详情页后，会看到更短、更旧的状态文案，难以确认当前任务属于哪个生成阶段、生成类型或结果类型。

本问题不只影响“已完成”状态，其他状态也需要同步补齐，例如排队中、运行中、失败、已停止、等待确认等。

## 目标行为

任务详情页应与生成页使用同一套状态展示规则。

状态展示应包含两层信息：

1. 主状态：任务当前生命周期状态。
2. 生成阶段或任务动作：当前或最后相关的生成步骤。

示例：

- `排队中 · 生成 FPA`
- `运行中 · 生成 FPA`
- `等待确认 · 生成 FPA`
- `已完成 · 生成 FPA`
- `失败 · 生成 FPA`
- `已停止 · 生成 FPA`

当无法取得步骤信息时，允许退化为单独状态文案，例如：

- `排队中`
- `运行中`
- `已完成`
- `失败`
- `已停止`

## 状态文案口径

详情页应优先复用生成页已有口径，而不是重新硬编码一套。

当前生成页 `Home.vue` 中的任务状态文案为：

| run state | 生成页文案 | 详情页目标文案 |
| --- | --- | --- |
| `idle` | `就绪` | 通常不出现在历史详情；如出现则显示 `就绪` |
| `queued` | `排队中` | `排队中` |
| `running` | `运行中` | `运行中` |
| `done` | `已完成` | `已完成` |
| `error` | `出错` | `出错` 或按生成页最终口径统一 |
| `cancelled` | `已停止` | `已停止` |

当前详情页 `TaskDetail.vue` 中的状态文案为：

| state | 当前详情页文案 | 需要调整 |
| --- | --- | --- |
| `queued` | `排队中` | 保持，并补齐步骤后缀 |
| `running` | `运行中` | 保持，并补齐步骤后缀 |
| `done` | `完成` | 改为 `已完成`，并补齐步骤后缀 |
| `error` | `失败` | 与生成页确认后统一，补齐步骤后缀 |
| `cancelled` | `已取消` | 与生成页统一为 `已停止`，补齐步骤后缀 |
| `closed` | `关闭` | 需要确认是否仍为有效任务状态 |

步骤文案应复用现有步骤定义：

| step key | 文案 |
| --- | --- |
| `basedata` | `读取基础数据` |
| `fpa` | `生成 FPA` |
| `spec` | `生成需求说明书` |
| `cosmic` | `生成 COSMIC` |
| `list` | `生成需求清单` |

## 辅助说明口径

生成页中状态徽标下方会显示 `runStateDetail`，来源为当前步骤的 `current_action`。

详情页如果展示状态辅助说明，应遵循同一规则：

- 优先显示当前或最后相关步骤的 `current_action`。
- 完成状态下，如果步骤提供了完成说明，则展示该说明。
- 无可用说明时不强行展示占位文案。
- 不为 FPA 写死辅助说明，避免影响 COSMIC、需求说明书、需求清单等任务。

## 推荐实现方案

### 1. 抽出共享状态格式化逻辑

新增或复用一个前端共享函数，负责从任务状态和步骤信息生成展示模型。

建议输出结构：

```ts
interface TaskStatusDisplay {
  label: string
  detail: string
  tone: 'idle' | 'queued' | 'running' | 'done' | 'error' | 'cancelled' | 'warning'
}
```

其中：

- `label`：用于徽标主文案，例如 `已完成 · 生成 FPA`。
- `detail`：用于辅助说明，例如 `FPA 工作量评估已生成`。
- `tone`：用于统一颜色和图标样式。

### 2. 统一步骤选择规则

生成页当前规则是：

1. 优先取 `running` 或 `waiting_input` 的步骤。
2. 否则从后往前取最后一个非 `pending`、有交付物或有 `current_action` 的步骤。
3. 找不到步骤时返回 `null`。

详情页应复用相同规则，确保“运行中”“等待确认”“完成”“失败”等状态都能拿到一致的步骤后缀。

### 3. 替换详情页本地状态映射

`TaskDetail.vue` 当前存在本地 `stateLabel(state: string)`：

```ts
function stateLabel(state: string) {
  const map: Record<string, string> = {
    queued: '排队中',
    running: '运行中',
    done: '完成',
    error: '失败',
    cancelled: '已取消',
    closed: '关闭',
  }
  return map[state] || state
}
```

实施时应将详情页状态卡片改为使用共享展示模型，避免继续维护这份与生成页不一致的映射。

### 4. 统一样式语义

详情页状态徽标颜色应与生成页同类状态保持一致：

- `queued`：警告色
- `running`：强调色
- `done`：成功色
- `error`：危险色
- `cancelled`：警告色
- `idle`：弱化色

如果详情页当前样式类与生成页差异较大，应优先保证状态颜色语义一致；布局可以保持详情页原有卡片结构。

## 验收标准

- [ ] 同一个 FPA 任务在生成页显示 `已完成 · 生成 FPA` 时，详情页也显示 `已完成 · 生成 FPA`。
- [ ] 详情页不再显示旧文案 `完成`。
- [ ] `cancelled` 状态在详情页与生成页统一为 `已停止`。
- [ ] `queued`、`running`、`done`、`error`、`cancelled` 状态均可显示对应步骤后缀。
- [ ] 无步骤信息时，详情页能退化显示单独状态，不出现 `undefined`、空点号或多余分隔符。
- [ ] FPA、COSMIC、需求说明书、需求清单等任务的步骤后缀按实际步骤动态显示，不写死为 `生成 FPA`。
- [ ] 生成页原有状态展示不发生回退。
- [ ] 详情页状态卡片在窄屏下不与“来源”“输入”“更新时间”等卡片内容重叠。

## 风险与注意事项

- 详情页历史数据中的状态字段可能与生成页运行态字段命名不同，实施时需要做状态归一化。
- `error` 在生成页为 `出错`，详情页当前为 `失败`，需要确认最终业务口径后统一。
- `closed` 状态是否仍然有效需要确认；如后端仍会返回，需定义展示文案和样式。
- 不要把 `生成 FPA` 写死到详情页；状态后缀必须来自步骤或任务类型。
- 如果历史任务缺少完整步骤数据，应保留合理退化展示。
